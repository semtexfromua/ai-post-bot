# Проєкт M4: AI-генератор постів для Telegram

## 1. **Вступ**

Ви приєдналися до команди стартапу, який прагне автоматизувати новинний Telegram-канал та вивести його на новий рівень.
Ваше завдання — створити сервіс, який не просто парсить новини, а використовує штучний інтелект (AI) для генерації яскравих, лаконічних та цікавих постів на основі зібраних матеріалів.
Публікація в Telegram-канал має відбуватися за розкладом, з можливістю ручного керування та моніторингу через API.

Вам належить інтегрувати кілька сервісів: парсинг новин (сайти та Telegram-канали), чергу завдань (task queue), генерацію постів за допомогою AI, публікацію через Telethon та реалізацію панелі керування джерелами.

Термін виконання — **2 тижні**. 4 вебінари.

## 2. **Функціональні блоки**

### **1. Збір та парсинг новин (сайти та Telegram-канали)**

- Збирати новини не лише із сайтів, а й з публічних Telegram-каналів (за допомогою Telethon).
- Для кожного джерела реалізувати окремий парсер.
- Запускати збір новин за розкладом (кожні 30 хвилин, Celery Beat).
- Кожна новина має містити: `title`, `url` (якщо є), `summary`, `source`, `published_at`.

------

### **2. Черга завдань та асинхронна обробка**

- Використання **Celery** та брокера (**RabbitMQ** або Redis) для обробки завдань (парсинг, генерація, публікація).
- Фоновий ланцюжок (pipeline): парсинг → фільтрація → генерація поста через AI → публікація.
- Усі "важкі" операції виносити в окремі таски (tasks) Celery.

------

### **3. AI-генерація постів**

- Використовувати OpenAI GPT-4 або іншу AI-модель через публічний API.
- На вхід подається: текст новини/summary/пост із Telegram. На виході: згенерований пост за заданим промптом (prompt).
- Налаштувати промпт: "Зроби короткий, цікавий опис новини для Telegram-каналу, додай емодзі (emoji), call to action".
- Обробка помилок API (rate limit, недоступність сервісу).
- Можливість вручну протестувати генерацію через API.

------

### **4. Фільтрація та відбір релевантних новин**

- Перед AI-генерацією фільтрувати новини за ключовими словами, мовою, джерелом.
- Гнучка система фільтрів (через налаштування або API).
- Виключення дублів (перевіряти за title/url/контентом).

------

### **5. Публікація в Telegram через Telethon**

- Після генерації AI-поста відправляти його у вказаний Telegram-канал (через Telethon).
- Перевіряти, чи не був цей пост уже опублікований.
- Логувати успішні публікації та помилки.

------

### **6. Керування джерелами та ключовими словами (адмін-панель через API)**

- REST API для додавання, редагування, видалення джерел новин (сайти, Telegram-канали).
- API для керування списком ключових слів та фільтрів.
- Ендпоінти (endpoints) для перегляду історії постів та логів помилок.

------

### **7. Документація API (Swagger)**

- Автоматична генерація через FastAPI (`/docs`).
- Задокументувати основні ендпоінти: новини, генерація, публікація, джерела, фільтри.

## 3. **Структура даних**

**Модель NewsItem:**

- `id` (uuid або hash)
- `title` (str)
- `url` (str, optional)
- `summary` (str)
- `source` (str)
- `published_at` (datetime)
- `raw_text` (str, якщо з Telegram)

**Модель Post:**

- `id`
- `news_id`
- `generated_text` (str)
- `published_at` (datetime)
- `status` (new/generated/published/failed)

**Модель Source:**

- `id`
- `type` (site/tg)
- `name`
- `url` (або username для TG)
- `enabled` (bool)

**Модель Keyword:**

- `id`
- `word`
